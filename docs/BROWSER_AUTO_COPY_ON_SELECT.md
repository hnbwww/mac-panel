# 浏览器自动复制功能

## 更新时间
2026-03-06 17:24

## 🎉 新功能：选中文字自动复制

### 功能说明

当你在网页上拖拽选择文字后，系统会**自动将文字复制到剪贴板**，并显示提示"✓ 文字已选中并复制到剪贴板"。

### 为什么需要这个功能？

**问题**：
1. 选中文字后，需要等待截图刷新（0.5秒）才能看到选中状态
2. 一滚动页面，选中状态就看不到了
3. 用户不知道文字是否真的被选中了

**解决方案**：
- 选中后自动复制
- 显示明确的提示消息
- 用户可以直接粘贴，无需担心选中状态丢失

### 使用方法

```
1. 打开浏览器，导航到一个有文字的网页
2. 在文字起始位置按下鼠标左键
3. 拖动到文字结束位置
4. 松开鼠标
5. ✅ 看到提示："✓ 文字已选中并复制到剪贴板"
6. 点击输入框，按 Ctrl+V
7. ✅ 文字被粘贴出来
```

### 技术实现

```typescript
// 鼠标松开 - 完成选择并自动复制
const handleMouseUp = (e: React.MouseEvent<HTMLImageElement>) => {
  if (!isSelecting || !wsRef.current) return;

  // 发送选择事件
  wsRef.current.send(JSON.stringify({
    type: 'selectText',
    data: {
      startX: selectionStart.x,
      startY: selectionStart.y,
      endX: endX * scaleX,
      endY: endY * scaleY,
      final: true
    }
  }));

  // 自动复制选中的文字
  setTimeout(() => {
    if (wsRef.current) {
      wsRef.current.send(JSON.stringify({
        type: 'copy',
        data: {}
      }));
      message.success('✓ 文字已选中并复制到剪贴板');
    }
  }, 100); // 稍微延迟，确保选择操作完成

  setIsSelecting(false);
  setSelectionStart(null);
};
```

### 执行流程

```
用户拖拽选择文字
    ↓
松开鼠标（触发 mouseup 事件）
    ↓
前端发送 selectText 消息到后端
    ↓
后端在真实浏览器中执行拖拽选择
    ↓
等待 100ms（确保选择完成）
    ↓
前端发送 copy 消息到后端
    ↓
后端执行 Ctrl+C（复制到剪贴板）
    ↓
前端显示成功提示
    ↓
用户可以随时粘贴（Ctrl+V）
```

### 优点

1. **无需担心选中状态丢失**
   - 即使滚动页面，文字也已在剪贴板
   - 不需要看到选中状态

2. **操作更简单**
   - 无需手动按 Ctrl+C
   - 选完即复制，一步到位

3. **明确的反馈**
   - 显示提示消息
   - 用户知道操作成功

4. **符合用户习惯**
   - 类似于某些应用的行为（如微信）
   - 选完即用

### 注意事项

1. **每次选择都会复制**
   - 如果再次选择其他文字，会覆盖之前剪贴板的内容
   - 这是正常的剪贴板行为

2. **延迟 100ms**
   - 确保选择操作完成后再复制
   - 避免复制失败

3. **提示消息**
   - 使用 Ant Design 的 message 组件
   - 自动在几秒后消失
   - 不阻塞用户操作
