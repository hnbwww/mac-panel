# 浏览器输入功能使用说明

## 更新时间
2026-03-06 17:20

## ⚠️ 重要说明

### 关于网页文字选择

**问题**: 用户反馈"网页文字选择、复制、粘贴功能不支持"

**解释**: 这些功能是**无法实现的**，原因是：

1. **浏览器画面是截图（PNG图片）**
   - 后端通过 Puppeteer 截取浏览器页面
   - 发送给前端的是 base64 编码的图片
   - 图片是静态的，无法选择上面的文字

2. **类似查看照片**
   - 你无法在照片上选择文字
   - 同样无法在浏览器截图上选择文字

3. **正确的使用方式**：
   - 先点击网页上的输入框（如 Google 搜索框）
   - 然后直接在前端输入文字
   - 前端会将键盘事件发送到后端
   - 后端在真实浏览器中执行输入操作

### 工作原理

```
用户操作流程：
1. 点击浏览器画面上的输入框（发送 click 事件到后端）
   ↓
2. 后端在真实浏览器中点击该输入框（输入框获得焦点）
   ↓
3. 用户在前端按键盘
   ↓
4. 前端捕获键盘事件，发送到后端
   ↓
5. 后端在真实浏览器中执行对应的键盘操作
```

## ✅ 支持的功能

### 1. 文字输入

**操作步骤**：
1. 点击浏览器画面上的输入框（如 Google 搜索框）
2. 直接输入文字
3. 文字会出现在后端真实浏览器的输入框中

**示例**：
```
1. 打开 Google (www.google.com)
2. 点击搜索框
3. 输入 "hello world"
4. 按 Enter 搜索
```

### 2. 删除文字

**支持的操作**：
- ✅ **Backspace** - 删除光标前的字符
- ✅ **Delete** - 删除光标后的字符

**操作步骤**：
1. 点击输入框，输入一些文字
2. 按 Backspace 或 Delete 键
3. 字符会被删除

### 3. 光标移动

**支持的操作**：
- ✅ **←** - 向左移动光标
- ✅ **→** - 向右移动光标
- ✅ **↑** - 向上移动（多行文本）
- ✅ **↓** - 向下移动（多行文本）

### 4. 复制粘贴

**复制操作**：
1. 在网页中选中文字（拖拽选择）
2. 按 **Ctrl+C** 或点击右键菜单"复制"
3. ✅ 文字已复制到真实浏览器的剪贴板

**粘贴操作**：
1. 点击输入框
2. 按 **Ctrl+V** 或点击右键菜单"粘贴"
3. 如果前端无法访问剪贴板（非 HTTPS），会弹出输入框
4. 输入要粘贴的文本，点击"粘贴"
5. ✅ 文字会被粘贴到输入框

**注意**：
- 后端真实浏览器的剪贴板和本地电脑的剪贴板是**独立的**
- 后端无法访问本地剪贴板（安全限制）
- 使用手动输入的方式粘贴文本

### 5. 其他快捷键

| 快捷键 | 功能 | 状态 |
|--------|------|------|
| **Ctrl+A** | 全选 | ✅ |
| **Ctrl+C** | 复制 | ✅ |
| **Ctrl+V** | 粘贴 | ✅ |
| **Ctrl+X** | 剪切 | ✅ |

## ❌ 不支持的功能

### 1. 在截图上直接选择文字

**原因**: 浏览器画面是图片，不是 HTML

**解决方案**:
- 先点击输入框
- 然后使用键盘快捷键操作

### 2. 查看后端浏览器的选中状态

**原因**: 只能看到截图，无法实时看到选中状态

**解决方案**:
- 等待下一次截图刷新（0.5秒）
- 可以看到操作的结果

### 3. 本地剪贴板同步

**原因**: 安全限制，后端无法访问本地剪贴板

**解决方案**:
- 使用弹出的输入框手动输入要粘贴的文本

## 🔧 技术实现

### 前端监听全局键盘事件

```typescript
useEffect(() => {
  const handleGlobalKeyDown = (e: KeyboardEvent) => {
    // 只在连接状态且有截图时处理
    if (!wsRef.current || !screenshot) return;

    // 避免在输入框中打字时触发
    const target = e.target as HTMLElement;
    if (target.tagName === 'INPUT' || target.tagName === 'TEXTAREA') {
      return;
    }

    // 处理功能键
    if (e.key === 'Backspace') {
      wsRef.current.send(JSON.stringify({
        type: 'keypress',
        data: { key: 'Backspace' }
      }));
    }

    // 处理普通字符
    if (e.key.length === 1) {
      wsRef.current.send(JSON.stringify({
        type: 'type',
        data: { text: e.key }
      }));
    }
  };

  document.addEventListener('keydown', handleGlobalKeyDown);
  return () => document.removeEventListener('keydown', handleGlobalKeyDown);
}, [screenshot]);
```

### 后端处理键盘事件

```typescript
async keypress(sessionId: string, key: string): Promise<void> {
  const session = this.sessions.get(sessionId);
  const { Input } = session.client;

  await Input.dispatchKeyEvent({
    type: 'keyDown',
    key: 'Backspace',
    code: 'Backspace'
  });

  await Input.dispatchKeyEvent({
    type: 'keyUp',
    key: 'Backspace',
    code: 'Backspace'
  });
}
```

## 常见问题

### Q1: 为什么无法在网页上选择文字？

**A**: 浏览器画面是截图（图片），无法在图片上选择文字。正确的方式是：
1. 先点击输入框
2. 使用键盘快捷键（Ctrl+A 全选，Ctrl+C 复制等）

### Q2: 为什么粘贴时弹出输入框？

**A**:
- 前端运行在 `http://192.168.0.7`（非 HTTPS）
- 浏览器禁止非 HTTPS 网站访问剪贴板
- 解决：使用输入框手动输入要粘贴的文本
- 或者：使用 HTTPS 部署前端

### Q3: 输入文字后为什么看不到立即反馈？

**A**:
- 截图刷新率是 2 FPS（每 2 秒刷新一次）
- 等待 0.5 秒后会看到更新
- 这是网络延迟导致的正常现象

### Q4: 为什么点击输入框后没有焦点指示？

**A**:
- 后端真实浏览器的输入框有焦点
- 但前端截图上看不到焦点指示
- 等待截图刷新后会看到光标

### Q5: 能否实现像本地浏览器一样的体验？

**A**:
- 无法完全一样，因为是通过截图+事件模拟
- 但可以通过以下方式改进：
  - 提高截图刷新率（增加带宽消耗）
  - 本地输入预览（复杂度高）
  - 使用视频流而非截图（需要改造架构）
